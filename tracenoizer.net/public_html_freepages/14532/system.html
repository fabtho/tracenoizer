- system -

Each package encapsulates all of its data model, library code, logic, adminstration pages and user pages in a single part of the file tree. Encapsulating everything about a package in one place also makes it much easier to distribute packages indepedently from the OpenACS itself. In order to make this work, we need a system that keeps track of the packages that have been installed in the server, where those packages have been installed, and a standard way to map URLs that a client sends to our server to the right page in the appropriate package. The APM The APM is used to create, maintain, and install packages. It takes care of copying all of the files and registering the package in the system.
