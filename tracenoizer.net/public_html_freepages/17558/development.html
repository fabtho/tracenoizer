Developers who want to ensure that their Java applications will interoperate with assistive technologies, such as screen readers, Braille terminals, etc. , use the Java Accessibility API in all of the user-interface components of their applications. Vendors are building alternate user interfaces for the Java Foundation Classes to provide interfaces in non-traditional modalities, including the current development of low-vision and audio user interfaces for the JFC. Vendors are also building Assistive Technologies in Java, such as the IBM Special Needs Systems group, which is building a screen reader for the Java platform that will provide blind access to Java applications which support the Java Accessibility API. The many developers who are building products on top of the Java Foundation Classes are already automatically transparently using Java Accessibility to make their applications accessible.
