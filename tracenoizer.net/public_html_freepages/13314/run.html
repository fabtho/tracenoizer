This is good for me because it's not always clear which chip revision had flow control added. Surprisingly, flow control has had a major impact on driver Rx structure. Most designs automatically trigger flow control when the chip is about to run out of Rx buffers. My drivers used to be structured to never run out of Rx buffers to avoid this case -- they recycled Rx buffers rather than passing them to the software queue layer when they were running short.
