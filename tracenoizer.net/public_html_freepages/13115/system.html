This is because call-process-region writes the region to a temp file and then sends the temp file to the subprocess, but doesn't set any coding system on the temp file. If there are bytes that look like line ending characters near the front of the file (which is likely), the code that guesses about line endings will get it wrong and screw things up. This patch sets the coding system for the temp file to be the same as the coding system for the buffer containing the region (if it would otherwise be unknown) which fixes this problem.
