SRM will use various methods to cope with the problems presented to programmers. It will offer caching of variables directly from supported scripting languages, and as they will be held in memory there will be considerable speed increases over a database-only approach. Another feature of SRM will be it's ability to handle all database connections persistently; this will offer significant advantages in a variety of situations where you are either using multiple supported scripting languages or want to be able to control the amount of connections spawned to your database server in a more effective way. SRM does not spawn a connection to your database server for each connection but decides whether it is worth it or if it is better to share another connection. .
