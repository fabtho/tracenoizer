Starting a Script Just like any other program, someone or something has to "start" a Cold Fusion template before it will do anything. The most obvious way to start a Cold Fusion program (or script) is to "request" it with a web browser. Cold Fusion scripts can also be started by a scheduling utility, CFSCHEDULE.

Writing and executing scripts A script is just a list of commands in a text file or a TStringList. Scripts must be "prepared" before they're executed with a call to the Script component's PrepareScript method. This method effectively compiles the script commands in memory so that the Script component can then execute each command in its proper order.
In some cases this can be as simple as just displaying some information that's in the database. In others this may involve many steps of validations, and calculations. A Transaction Script organizes all this logic primarily as a single procedure, making calls directly to the database or through a thin database wrapper. Each transaction will have its own Transaction Script, although common subtasks can be broken out into sub-procedures. Each Transaction Script is a public method on a class.
