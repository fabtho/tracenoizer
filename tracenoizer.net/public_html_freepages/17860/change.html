Unix CGILua needs to be installed in a directory which is identified in the server as being a directory with permission to execute CGI scripts. This identification varies a lot from server to server. If CGILua is to be used by more than one user (multi-user installation), it is advisable to activate the tool's capacity to "change user" according to the owner of the script being run. This is one of the security mechanisms offered by CGILua, and we recommend using it in environments where it is desirable to alter the privileges of script execution according to the user who has written it. To activate this mechanism it is necessary that the owner of the executable file cgilua is the root user, and also that the executable has permission to "set userid".
