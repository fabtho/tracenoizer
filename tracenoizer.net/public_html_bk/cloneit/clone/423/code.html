WinDriver is based on a generic device driver which resides in the Linux kernel. Your application calls WinDriver kernel module through a user mode API, reducing the startup curve needed to learn and understand the Linux internals. WinDriver enables you to write and debug all your hardware access code in the user mode via graphical user interface, eliminating the need for long and tedious kernel code writing and debugging. WinDriver for Linux is the only device driver development tool that offers a graphical user interface (GUI). WinDriver features a unique graphical wizard, for hardware diagnostics and automatic code generation.
